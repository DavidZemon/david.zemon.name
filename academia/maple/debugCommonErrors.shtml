Debugging: Common Errors

The built-in Help
Maple's built-in help is, unlike most built-in help databases, very useful. Access it one of three ways: Help -> Maple Help, Cntrl + F1, or my preferred way, type "?____" where the blank space is any Maple command or math term you can think of. For instance, to learn some advanced features of the plot command, try typing and executing ?plot.
"Error, unable to match delimiters"
By far, the most common mistake you'll make is with parenthesis. If Maple returns this error, then you've made a mistake with your parenthesis, curly brackets, or square brackets. Go back through your code and re-check these items.
Tip: placing your cursor to the right of a closing bracket or left of an opening bracket will underline all text between that bracket and it's respective opening or closing bracket.
Maple returns wrong answer & contains variable name rather than numbers
This most likely occurred because you did not start from your restart command. Maple does not have a set order which it follows. If you type a set of commands in the order you wish them to be carried out, YOU are in charge ensuring they are carried out in that order. You can not simply excecute the final line and assume all lines above it will be executed. Either begin execution at the necessary restart command and proceed executing all commands beneath it, or press the "!!!" button at the top, which will execute all commands from top to bottom in your entire worksheet. This can be a good way to ensure you have everything in order with restarts where they need to be.

Copying and Pasting
In short, don't. Copying and pasting while using math-type can screw you up and it's nearly impossible to find where Maple messed up. Do not attempt this if A) you're anything but a Maple pro, or B) you're tight for time and can't risk wasting any of that precious commodity on debugging.
However, if you, for some reason, decide to try it despite my warnings, here are my finding so far. The problem I've run across when using copying & pasting (and I have not figured out why this happens sometimes, and not all the time) is that Maple sees the pasted data as a separate command from the rest of your line. This is fairly easy to spot once you're looking for it because there will be a point in your line where you are unable to backspace any further. If you find that point, delete the entire line and try again, without copying and pasting. Assign a variable to the data you want to use, or just retype it all by hand.
"RootOf(_Z - BunchOfCrap)"
Try using evalf on this. Sometimes it works. Sometimes it doesn't. If it doesn't, chances are you did something wrong. Perhaps a math major would be able to tell you what this actually means - all I know is that there is some very complex function out there and my answers shouldn't look like it. Recheck your work and look for typos.
Solve returns nothing
The most common reason I've run across this error is because I'm trying to solve a system of equations without entering all of my variables in the second argument.
restart
f := sin(x)+exp((1/100)*y)
g := sin(y)+exp(-(1/100)*x)
evalf(solve({f = 1, g = 0}, x))
In this case, you need to add the second variable to your solve command - Maple won't solve for only x.

evalf(solve({f = 1, g = 0}, {x, y}))
"Warning, solving for expressions other than names or functions is not recommended."
By "not recommended" Maple actually means there is no way on Earth it's going to let you solve for that. Check your second argument and make sure you didn't make a typo. If that doesn't fix it, then you need to approach the problem a different way because Maple is not accepting your second argument.
